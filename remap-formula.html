Core utility formula

Remap one range into another

Use remap when one gameplay value needs to drive another scale, like health to UI fill or speed to effect intensity.

At a glance

OutMin + ((Value - InMin) / (InMax - InMin)) * (OutMax - OutMin)

Remap helps you take a value that makes sense in one range and convert it into another range without losing the relative position.

Simple explanation

What the formula means

Remap finds where the input sits inside its source range, then puts it at the same relative point inside the target range.

Code example

float result = outMin + ((value - inMin) / (inMax - inMin)) * (outMax - outMin);

When to use it

Common game-dev uses

  • Convert health from 0 to 250 into a UI fill from 0 to 1.
  • Turn trigger input from 0 to 1 into movement speed from walk to sprint.
  • Map progress, XP, or charge values into a custom visual scale.

Variables

What each variable represents

  • Value: The number you want to convert.
  • InMin: The low end of the source range.
  • InMax: The high end of the source range.
  • OutMin: The low end of the output range.
  • OutMax: The high end of the output range.

Constants

Fixed values in the formula

  • No fixed constants. The behavior comes from the values you provide.
