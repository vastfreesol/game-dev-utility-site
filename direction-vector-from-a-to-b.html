2D formulas

Direction vector from A to B

Use this when you need the direction an object should move, aim, or face in 2D space. It answers "which way?" before you worry about speed, pathfinding, or acceleration.

The formula

Direction = normalize(B - A)

Why you would use it

A raw distance tells you how far away the target is. A direction vector tells you where to go. This page turns the gap between two points into both a raw vector and a normalized movement direction.

How to think about it

Separate direction from speed

A direction vector is most useful when you normalize it first. That gives you a clean heading with magnitude 1, which means you can multiply it by any speed value later without changing its direction.

Game-dev use cases

  • Move an enemy straight toward the player.
  • Fire a projectile from one object toward another.
  • Work out knockback or steering directions.

Variable breakdown

What each piece is doing

  • B - A: the raw vector pointing from the source point to the target point.
  • dx, dy: the horizontal and vertical change between the two positions.
  • normalize(...): divides the vector by its length so the final direction has magnitude 1.

Common mistake

A zero-length vector cannot be normalized

If both points are the same, there is no direction to travel. In code, this usually means you need a guard clause before normalizing the vector.
