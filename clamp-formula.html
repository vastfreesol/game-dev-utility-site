Clamp a value into a safe range

Use clamp when a stat, slider, camera value, or gameplay number must stay between a minimum and a maximum.

At a glance

Clamp(Value, Min, Max)

Clamp is one of the most common helper formulas in game code. It protects values from drifting too low or too high.

Simple explanation

What the formula means

Clamp checks whether the value is too low or too high. If it is outside the allowed range, it snaps to the nearest edge.

Code example

float result = Math.Clamp(value, min, max);

Pseudocode

if value < min
    result = min
else if value > max
    result = max
else
    result = value

When to use it

Common game-dev uses

  • Keep health, stamina, or mana from going below zero or above the current cap.
  • Stop UI percentages from showing values outside 0 to 1 or 0 to 100.
  • Limit camera pitch, speed modifiers, or sensitivity settings to safe values.

Variables

What each variable represents

  • Value: The number you want to control.
  • Min: The lowest result you will allow.
  • Max: The highest result you will allow.

Constants

Fixed values in the formula

  • No fixed constants. The behavior comes from the values you provide.
