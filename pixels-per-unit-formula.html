2D formulas

Pixels per unit

Use this when you want to turn sprite dimensions into a clean in-engine scale. It helps you decide how many texture pixels should map to one world unit before you import or place 2D art.

The formula

PPU = SpritePixels / WorldUnits

Why you would use it

PPU is one of the quickest ways to keep sprites, props, and tiles consistent. Instead of eyeballing scale in engine, you can choose the world size you want and let the utility give you the matching import value.

How to think about it

Choose world scale first, then derive import scale

It is usually easier to decide how big something should feel in the world before you worry about import settings. Once you know that target size, PPU becomes a clean conversion rather than a guess.

Game-dev use cases

  • Set consistent Unity or Godot import scale across characters and props.
  • Align sprites with tile sizes and collision metrics.
  • Keep hand-made art and generated assets feeling like they belong together.

Variable breakdown

What each piece is doing

  • SpritePixels: the source size of the sprite, usually its height for character-like assets.
  • WorldUnits: how much in-world space you want the sprite to occupy.
  • PPU: how many pixels map to one game unit once imported.

Common mistake

Do not mix art scale and camera scale by accident

Changing PPU changes how art maps into world units, not how the camera sees the world. If something still feels wrong after setting PPU, the issue may be camera setup, zoom, or tile metrics instead.
