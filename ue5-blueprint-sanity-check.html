How to read this

  • Official target means Epic documents the owning class or library for the node.
  • Directly on self means the Blueprint type naturally owns that target or inherits it.
  • Via references means you can still use the node when you hold the right object reference, even if the Blueprint itself is not the target.
  • Inference is where this page connects the docs into a practical "where should I put this?" recommendation.

Practical framing

How to think about node availability

In Unreal, many Blueprint nodes are tied to a target class or a static library instead of a neat "this Blueprint can use it" checkbox. That is why a sanity check is helpful: it turns target metadata, parent classes, and library ownership into a faster recommendation for where the graph probably belongs.

The result is intentionally practical rather than absolute. A node can often be called from many places once you hold the right reference, but the cleanest home for it is usually narrower.
