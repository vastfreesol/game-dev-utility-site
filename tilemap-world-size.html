2D formulas

Tilemap world size

Use this when you need to know how large a level will be before exporting, framing cameras, or budgeting memory. It turns tile size and grid counts into a final map footprint.

The formula

MapWidth = TileWidth * Columns; MapHeight = TileHeight * Rows

Why you would use it

Tilemaps are easy to reason about in rows and columns, but production decisions usually need real dimensions. This page gives you the final pixel footprint, total tile count, and an optional world-unit conversion in one place.

How to think about it

Rows and columns become real space quickly

A tilemap can feel small in editor terms but become quite large once you multiply out the tile dimensions. This is why it helps to check total size early, especially for camera framing, export size, and world streaming decisions.

Game-dev use cases

  • Estimate final map size before exporting a tilemap or background layer.
  • Plan camera framing and level boundaries.
  • Compare multiple map sizes while scoping a level.

Variable breakdown

What each piece is doing

  • TileWidth, TileHeight: the size of one tile in pixels.
  • Columns, Rows: how many tiles the map uses horizontally and vertically.
  • PixelsPerUnit: optional conversion value if you want the result in world units too.

Common mistake

Tile size and tile count are easy to mix up

A map with 100 columns of 32px tiles is 3200px wide, not 100px wide. Keeping the dimensions and counts separate is the easiest way to avoid bad camera and export estimates.
